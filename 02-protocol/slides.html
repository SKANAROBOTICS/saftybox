Context

Where This Fits

The safety system guarantees motor power is removed unless a genuine, current authorization is present. It is a chain — break any link and motors lose power.

Protocol's job

  • Continuously prove authorization to run — the lease
  • Over an extremely low-bandwidth, high-latency acoustic link
  • Enable only on a genuine, fresh, intended heartbeat

Main challenge

  • Immune to accidental triggering — random noise must not authorize
  • Immune to record/replay — old captured response must not re-arm
  • Works over a link that drops, reorders, and delays
Resistance to a malicious adversary is explicitly out of scope. The threat model is accidental-fault-safe (IEC 61508), not adversary-proof (IEC 62443).
Design

Packet Format — 3 Bytes Each Direction

Minimum acoustic payload. Both directions fit in a single 3-byte frame.

Robot → Home status · 4 bits nonce · 20 bits = 24 bits = 3 bytes
Home → Robot n · 4 bits R · 20 bits = 24 bits = 3 bytes

Challenge (Robot → Home)

  • nonce — 20-bit random, unique within the robot's active buffer
  • status — 4-bit robot state carried free in the spare nibble

Response (Home → Robot)

  • n — 4-bit lease exponent; grants lease = 2ⁿ seconds
  • R — truncate₂₀( MAC_key( nonce ‖ n ) )
n is bound inside the MAC. A captured short-lease response cannot be presented as a long-lease one — the MAC fails for any other n.
Protocol Flow

Robot Side — Challenge & Lease Validation

1
Every T seconds, generate a fresh 20-bit nonce from the CSPRNG. Check it is not already in the buffer (collision prob. K/2²⁰, negligible). Append to circular buffer; displace oldest if full.
2
Pack [status:4 | nonce:20] and transmit over the acoustic link.
3
On receiving any 3-byte reply (n, R): prune expired nonces first, then iterate buffer newest-first — compute truncate₂₀(MAC_key(nonce_i ‖ n)) and compare to R.
4
First match → consume nonce_i from buffer, then kick the watchdog, grant lease 2ⁿ seconds. No match across full buffer → discard silently.
5
If the watchdog is not kicked before the current lease expires → relays open. Fail-safe.
Critical ordering: nonce must be consumed from the buffer before the watchdog kick is issued. A second delivery of the same R cannot kick a second time.
Protocol Flow

Home Side — Stateless Response

1
Receive a 3-byte challenge from the robot.
2
Operator's mode selection determines n (lease exponent).
3
Compute R = truncate₂₀( MAC_key( nonce ‖ n ) ).
4
Transmit [n:4 | R:20] — 3 bytes.
Home is stateless beyond the current received nonce. The robot's circular buffer absorbs all link latency and reordering — home just responds to whatever nonce it receives.

Nonce expiry (max_age)

Each buffer slot stores (nonce, expires_at). A nonce is only eligible if current_time < expires_at. This prevents pathologically delayed responses (TCP-tunneled, buffered) from being accepted, regardless of buffer depth.

Security Properties

Why This Works

K/2²⁰ False-accept prob per frame
1-in-131k For K=8 (24-bit MAC)
2ⁿ sec Lease, HW-capped
  • Replay immunity — nonce changes every T seconds; consumed-on-match means any captured (n, R) is useless once the nonce is gone.
  • Buffer uniqueness — pre-send uniqueness check ensures all K active nonces are distinct, eliminating birthday collisions by construction.
  • Lease authentication — n is bound inside the MAC; a short-lease capture cannot be presented as a long lease.
  • Wrong-key rejection — a unit without the correct key cannot produce a valid R.
  • Unauthenticated revoke — stop/revoke carries no MAC by design; it can only drive toward OFF (always safe). Do not add auth to the revoke path.
  • Fail-safe — operator stops, link drops, firmware hangs, power sags → lease expires → relays open.
Implementation

Entropy Source (Resolved)

Prototype — Teensy 4.x / i.MX RT1062

  • Hardware TRNG: FIPS 140-2 tested, free-running ring oscillators
  • Access via TRNG_ENT0–ENT15; poll TRNG_MCTL_ENT_VAL

Production MCU Criteria

  • Hardware TRNG is a hard selection criterion
  • Standard on STM32 F4/L4/H7/U5, Nordic nRF52/53, NXP i.MX RT

Boot Pattern (both)

1
Block until TRNG ready flag set (<10 ms)
2
Collect 32 bytes; seed ChaCha20 or AES-CTR-DRBG CSPRNG; clear TRNG output from RAM
3
Generate all nonces from CSPRNG; reseed from TRNG every ~1 hour on long missions
4
Hard reset: block nonce generation until TRNG ready + reseeded. Never seed from a fixed value.
Threat Analysis

Two Threat Models — One Boundary

In Scope — Accidental Fault (IEC 61508)

No intelligent adversary. Threats from entropy, physics, firmware bugs, channel noise, power failures.

  • Random-frame false acceptance
  • Accidental replay (multipath, ARQ retransmit)
  • Nonce entropy failure (stuck RNG, bias)
  • Buffer window / latency mismatch
  • Firmware hang → watchdog starvation
  • Key corruption in flash
  • Cross-n accidental collision
  • Post-reset buffer state

Out of Scope — Adversarial (IEC 62443)

Intelligent, motivated attacker. Correctly excluded. Documented in Safety Case.

  • Key extraction (JTAG, side-channel)
  • Man-in-the-middle attack
  • Deliberate timed replay
  • Brute-force forgery (16M crafted frames)
  • Deliberate acoustic jamming
  • Multi-UUV cross-routing (deliberate)
  • Side-channel on MAC computation
  • Supply-chain key injection
24-bit MAC adequacy: Cryptographically weak against an adversary (NIST SP 800-224 min is 32 bits). For the accidental-fault scope, the question is only: what is the probability that a randomly corrupted acoustic frame passes? These are entirely different calculations.
Threat Analysis · Findings

Key Gaps Identified

False-Accept Rate — Not Closed

The 1-in-131k figure is per frame, not per hour. The safety-case quantity is:

P(false/hr) = F_frames/hr × K×16 / 2²⁴

At F=3,600 frames/hr → one false accept every 36 hours. Must be closed against integrity target before deployment.

Nonce Width Too Narrow

Current spec is 20-bit nonce. Birthday collision in buffer of K=8:

  • W=20 bits: ~1-in-37k collision — marginal
  • W=32 bits minimum required
  • Wider nonce needs 4-byte packet (currently 3-byte)

Modem Dedup Responsibility — Unassigned

Anti-replay works only if the modem delivers each physical burst at most once at the application layer. If modem firmware performs link-layer ARQ and surfaces two deliveries, dedup must happen below the comparison loop. Ownership not assigned.

Archive MAC Width Conflict

archive/DESIGN.md specifies a 2-byte (16-bit) MAC; current spec is 3-byte (24-bit). For K=8, a 16-bit MAC gives ~1-in-512 odds per frame — unacceptable. Archive must be formally marked superseded.

Threat Analysis · Action Items

Must Resolve Before Implementation

  1. Nonce width W — minimum W=32 bits (4 bytes). Current 20-bit spec is insufficient for birthday-collision resistance. Cannot remain TBD.
  2. Entropy source — hardware RNG qualification or qualified CSPRNG with specified seeding after every power cycle. Replay immunity is not formally established without this.
  3. Per-hour false-accept budget — requires measured or bounded acoustic spurious frame rate. Close against the integrity target; upgrade to 32-bit R if F > ~0.1 frames/sec.
  4. Hardware lease ceiling — personality plug max n value must be defined and enforced in hardware before deployment.
  5. Archive DESIGN.md — formally mark as superseded to prevent implementation confusion over the 2-byte vs. 3-byte MAC discrepancy.
  6. Revoke path — document unauthenticated revoke as an intentional architectural constraint, not an omission. Future maintainers must not add auth to this path.
  7. Consume-then-kick ordering — specify as atomic in firmware requirements. Kick before consume allows a second delivery of the same R to re-arm.
  8. Modem deduplication — assign responsibility (modem layer vs. application layer) explicitly.
  9. Post-reset buffer state — buffer must be cleared on every reset path; behavior documented in the state machine.
  10. Buffer-full eviction — document silent displacement as expected behavior, not an error condition that could stop nonce generation.

Summary

Where We Stand

What's solid

  • Core challenge-response architecture is sound
  • Consume-on-match anti-replay is correct (IPsec pattern)
  • Home stateless design eliminates sync faults
  • Unauthenticated revoke is intentional and safe
  • Entropy source resolved for prototype MCU
  • n bound inside MAC prevents lease substitution

Blockers before implementation

  • Nonce width — must be 32 bits minimum
  • Entropy source — must be formally qualified
  • False-accept budget — must be closed per hour against integrity target
  • HW lease ceiling — personality plug max n must be defined
  • Archive conflict — 16-bit vs 24-bit MAC must be resolved
The protocol design is architecturally correct for the accidental-fault scope. The five blockers above are parameter and documentation gaps, not fundamental design flaws. Clear them and implementation can begin.
